Help with {% if create %}creating a new{% else %}editing a{% endif %} role

Custom roles can be built upon the limited, non-Administrative "User" role, to give specific entities just enough control in the application.
A user can be assigned a specific Role in this Edit Role page.
NOTE: Only administrative roles("Administrator", "Operator") have access to all domains (including the removal of them). Non-administrative users cannot have this ability.

Name is the name of the role. It cannot be edited after creation.
Description is a user friendly name for this role.
Access Control allows an Administrator or Operator to quickly assign the new Role to multiple users. Users removed from the right column will be given the default "User" Role.
Operators cannot assign a role to users who are either Adminstrators or Operators.
Allow DNSSEC configuration selects whether users who have this Role can toggle DNSSEC in their domains.
Allow History access for this role toggles whether users who have this Role can view History records for the domains they have access to.
Allow Domain creation for this role toggles whether users who have this Role can create new Domains.
Note that domains created by non-administrative users must belong to an account. Therefore, users who are not members in an account cannot create a new domain.
Allow Domain removal for this role toggles whether users who have this Role can remove Domains they have access to.
Allow DNSSEC configuration for this role grants users who have this Role Read or Write access to a specific DNS Record type, in all Forward, or all Reverse Domains.
In order to have Write access to a Record, a user must also have Read access to the same Record.
Allow to view and edit all domains for this role enables read access for all the domains. With this option toggled on, a non-administrative user assigned to this role, can only view and edit the domains.
This setting does not give the user Domain Removal access and History access for all domains.
Instead, they must be given access to the domain explicitly to use these features.
{% if role.name == 'Operator' %} Allow to edit roles option, permits users who are Operators, to modify the Role privileges of non-Administrative roles.
{% endif %} {% if role.name == 'Operator' %} NOTE: For the Operator role, the options

  • Allow History access for this role
  • Allow to view and edit all domains for this role
are enabled by default and cannot be toggled off. {% endif %}
